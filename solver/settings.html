SettingsÂ¶

These settings control how SCS behaves during a solve. They are set in the ScsSettings struct.

Name

Type

Description

Permitted values

Default

normalize

scs_int

Whether to perform heuristic data rescaling.

True/False

1

init_scale

scs_float

Initial dual scale factor (the important one).

\((0, \infty)\)

0.1

adaptive_scaling

scs_int

Whether to heuristically adapt dual scale through the solve.

True/False

1

rho_x

scs_float

Primal scale factor (less important one).

\((0, \infty)\)

1e-6

max_iters

scs_int

Maximum number of iterations to run.

\(\mathbf{N}\)

1e5

eps_abs

scs_float

Absolute feasibility tolerance.

\((0, \infty)\)

1e-4

eps_rel

scs_float

Relative feasibility tolerance.

\((0, \infty)\)

1e-4

eps_infeas

scs_float

Infeasibility tolerance (primal and dual).

\((0, \infty)\)

1e-7

alpha

scs_float

Douglas-Rachford relaxation parameter.

\((0, 2)\)

1.5

time_limit_secs

scs_float

Time limit for solve run in seconds (can be fractional). 0 is interpreted as no limit.

\([0, \infty)\)

0

verbose

scs_int

Whether to print solver output to stdout.

True/False

1

warm_start

scs_int

Set to True if you initialize the solver with a guess of the solution (see below).

True/False

0

acceleration_lookback

scs_int

How much memory to use for Anderson acceleration. More memory requires more time to compute but can give more reliable steps. 0 disables it.

\(\mathbf{N}\)

0

acceleration_interval

scs_int

Run Anderson acceleration every this number of iterations.

\(\mathbf{N}\)

1

write_data_filename

char *

If this is set the problem data is dumped to this filename.

Any filename

NULL

log_csv_filename

char *

If this is set SCS will write csv logs of various quantities through the solver. Doing this makes the solver much slower.

Any filename

NULL
